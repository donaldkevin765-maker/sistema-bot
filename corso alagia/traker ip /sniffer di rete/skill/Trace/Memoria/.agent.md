# 🤖 MEMORIA — Istruzioni per Agenti AI

> Ogni agente Sisyphus DEVE leggere questo file quando il task riguarda conoscenza, memoria, progetti.
> **LETTURA OBBLIGATORIA:** Anche `MANIFESTO.md` (filosofia) e `PROTOCOLLO.md` (regole scrittura).

---

## Cos'è

Knowledge graph markdown-based in **due reami** + **4 agenti autonomi** che lo mantengono vivo.

| Reame | Path | Contenuto |
|-------|------|-----------|
| **IO** | `personale/nodi/` | Skills, apprendimenti, interessi personali |
| **AZIENDA** | `azienda/nodi/` | Progetti, sistemi, strumenti, decisioni |

---

## Sistema a Agenti

La MEMORIA non è statica: **4 agenti Python** girano in autonomia (ogni giorno via launchd) e:

| Agente | Cosa fa | Output |
|--------|---------|--------|
| 🔍 **Scanner** | Verifica integrità nodi, orfani, anomalie | `_inbox/scanner-*.md` |
| 🔗 **Connector** | Trova connessioni non ovvie tra nodi | `_inbox/connector-*.md` |
| 🧠 **Synth** | Produce insight e pattern emergenti | `_inbox/synth-*.md` + `_auto/insights/` |
| 📊 **Observer** | Metriche, trend, salute del grafo | `_inbox/observer-*.md` + `_auto/reports/` |

### Ciclo completo
```bash
./_bin/agents                          # Tutti e 4 + evolve.py
./_bin/agents --agent connector        # Solo connector
./_bin/agents --skip-evolve            # Salta aggiornamento indici
```

### Autonomo (launchd)
Timer installato che esegue `_bin/agents` ogni giorno alle 08:00.
```bash
./script/install_launchd.sh            # Installa timer
launchctl start com.sisyphus.memoria   # Esegui subito
```

## Comunicazione Agenti ↔ Sisyphus

**La comunicazione segue due canali distinti a seconda del tipo di agente:**

### Canale 1 — Task diretto (Sisyphus → agente via `task()`)

Usato quando Sisyphus chiama un agente della flotta (Alpha/Beta/Delta/Zeta/Eta/Kappa).

```
Sisyphus → task(prompt) → agente → risultato diretto → Sisyphus
```

- La comunicazione è sincrona: Sisyphus invia, l'agente risponde subito
- Se l'agente ha un dubbio sul task, lo **scrive nel risultato**, non in `_inbox/`
- Sisyphus legge subito, chiarisce (se serve con FASE 0), e rilancia l'agente

### Canale 2 — Inbox (agenti autonomi → Sisyphus)

Usato dagli agenti Python autonomi (Scanner/Connector/Synth/Observer) che girano via `./_bin/agents` o launchd, senza Sisyphus in ascolto.

```
Agente autonomo → _inbox/ → (tempo) → Sisyphus legge in sessione → decide
```

Sisyphus legge l'inbox all'inizio di ogni sessione (vedi regola #4) e decide se:
- Promuovere suggerimenti ad archi
- Creare nuovi nodi
- Riorganizzare sezioni

Formato messaggio inbox:
```yaml
---
type: agent-report
agent: connector
timestamp: 2026-07-20T21:30:00
status: ok
---
---

### Protocollo scrittura in memoria (regola per TUTTI gli agenti)

**👉 Il protocollo completo è in `PROTOCOLLO.md` — DEVE essere letto da ogni agente prima di scrivere.**

Versione sintetica (7 gate):

```
[Gate 1] GIÀ ESISTE?     → query.py --cerca. Se sì: aggiorna, non duplicare
[Gate 2] SERVE TRA 6M?   → No → _tasks/ | Dubbio → _inbox/
[Gate 3] STA IN UN NODO?  → Aggiorna esistente, non crearne uno nuovo
[Gate 4] GRAFO GIUSTO?    → personale o azienda? Tipo dalla tassonomia
[Gate 5] FONTE SICURA?    → Certa → OK. Speculativa → _inbox/
[Gate 6] DATI SENSIBILI?  → Mai. Segnala immediatamente.
[Gate 7] È ORFANO?        → connesso_a obbligatorio (almeno 1 nodo)
```

**Se un gate blocca, l'agente NON scrive in nodi/ — scrive in `_inbox/` come candidata.**

**Chi scrive cosa:**

| Azione | Agente | Comando |
|--------|--------|---------|
| Creare nodo tecnico | Delta | `memoria_writer.py --grafo azienda ...` |
| Creare learning | Delta | `memoria_writer.py --grafo personale ...` |
| Aggiornare nodo | Eta (fix) / Delta (contenuto) | `memoria_writer.py --update ...` |
| Connettere nodi | Zeta | `memoria_writer.py --connect ...` |
| Segnalare anomalia | Qualsiasi | Scrive in `_inbox/` |
| Archiviare | Kappa | Solo dopo approvazione Alpha |
| Diagnostica PC | Epsilon | `./_bin/pc status` |
| Pulizia file PC | Epsilon | `./_bin/pc clean` |
| Trova file vecchi | Epsilon | `./_bin/pc old <giorni>` |
| Eseguire comando | Epsilon | `./_bin/pc exec <comando>` |

### Comunicazione Sisyphus ↔ Utente

**Prima di qualsiasi azione, Sisyphus deve capire bene cosa vuole l'utente.**

Se la richiesta non è chiarissima, Sisyphus attiva la **FASE 0 — CHIARIMENTO** (descritta in `PROTOCOLLO.md`):

```
Richiesta utente
     │
     ▼
┌─────────────────────────────┐
│ È chiarissima?              │
└─────────────────────────────┘
     │ No                              │ Sì
     ▼                                 ▼
┌──────────────────────┐    ┌──────────────────┐
│ FASE 0: Chiedi       │    │ Vai a FASE 1     │
│ Domande mirate       │    │ (7 Gate + Write) │
│ (dirette/indirette)  │    │                  │
│ + esempi se utile    │    │                  │
└──────────────────────┘    └──────────────────┘
     │
     ▼ (utente risponde)
┌──────────────────────┐
│ Chiaro? → Procedi    │
│ Ancora dubbio?       │
│ → Chiedi ancora (max │
│   3 turni, poi agisci│
│   con la migliore    │
│   interpretazione)   │
└──────────────────────┘
```

**Regole:**
- Dirette: quando manca un dettaglio tecnico preciso
- Indirette: quando vuoi capire l'intento senza influenzare
- Esempi: sempre utili dopo una spiegazione astratta
- Mai più di 3 domande per turno
- Se dopo 3 tentativi non è ancora chiaro → procedi con la migliore interpretazione

### Come Sisyphus delega

Quando arriva una richiesta complessa e CHIARA dall'utente:

```
1. Sisyphus analizza la richiesta
2. Valuta se è tutto chiaro (se no → FASE 0 prima)
3. La spezza in sotto-task (se serve più agenti)
4. Per ogni sotto-task, Sisyphus chiama task() con category appropriato
5. prompt = ruolo_agente + task_specifico + contesto + formato_output + regola_scrittura
6. Se più agenti in parallelo: chiamarli in background
7. Sisyphus colletta risultati e sintetizza risposta
```

Esempio:
```python
task(
    category="general",
    prompt="Sei **Beta**, il Ricercatore della flotta. "
           "Cerca informazioni su [argomento]. "
           "Prima di salvare, verifica con query.py se già esiste. "
           "Se è nuova e utile, crea nodo con memoria_writer.py. "
           "Restituisci risultati con fonti."
)
```

### Chiamata diretta

```bash
./_bin/fleet agent beta "cerca info su Playwright anti-detection"
./_bin/fleet list
./_bin/fleet status
```

## 👑 Sisyphus Scope — Cosa fa Sisyphus, cosa fanno gli agenti

### Principio
I 7 agenti hanno domini specializzati. **Tutto ciò che non rientra in quei domini, lo fa Sisyphus.**

### Cosa Sisyphus fa SEMPRE da solo (mai delegato)

| Task | Motivo |
|------|--------|
| **Sintesi multi-agente** | Combinare Beta+Gamma+Delta richiede visione d'insieme |
| **Promozione da inbox** | Decidere cosa promuovere da candidato a nodo serve contesto umano |
| **Risposta all'utente** | Gli agenti NON parlano mai con l'utente — solo Sisyphus |
| **Scartare informazione** | Decidere cosa NON memorizzare è più delicato che scrivere |
| **Riorganizzazione strutturale** | Spostare sezioni, rinominare grafi, cambiare tassonomia |
| **Merge/cancellazione nodi** | Kappa propone, Sisyphus approva ed esegue |
| **Nodi personali profondi** | Vita, obiettivi, relazioni in `personale/` — solo Sisyphus |
| **Fix errori critici** | Frontmatter rotto, grafo corrotto, script che non parte |

### Cosa Sisyphus fa DELEGANDO agli agenti

| Richiesta utente | Agente | Ruolo di Sisyphus |
|------------------|--------|-------------------|
| "Cerca info su X" | Beta | Lo lancia, controlla risultato |
| "Analizza trend" | Gamma | Lo lancia, riceve insight |
| "Aggiungi progetto Z" | Delta | Lo lancia, verifica nodo |
| "Collega X a Y" | Zeta | Lo lancia, approva arco |
| "Controlla qualità" | Eta | Lo lancia, applica fix |
| "Archivia nodi vecchi" | Kappa | Lo lancia, approva cancellazioni |
| "Pulisci file sul PC" | Epsilon | Lo lancia, valuta risultati |
| "Trova file vecchi" | Epsilon | Lo lancia, decide se cancellare |
| Task complesso multi-dominio | Alpha + altri | Sisyphus → Alpha → spezza → agenti → Alpha sintetizza → Sisyphus |

### Fallback: quando un task non ha un agente designato

```
1. È urgente?       → Sisyphus lo fa direttamente
2. È complesso?     → Sisyphus lo spezza in sotto-task che gli agenti possono fare
3. È generativo?    → Chiama Delta (l'architetto, più versatile)
4. È esplorativo?   → Chiama Beta (ricercatore, punto di partenza)
5. Residuo non assegnabile? → Sisyphus lo fa direttamente

Regola pratica:
  Visione d'insieme → Sisyphus
  Un solo dominio   → Agente specializzato
  Più domini        → Sisyphus coordina, agenti eseguono
  Non sai a chi?    → Sisyphus lo fa
```

## 🔄 Miglioramento Continuo

La flotta si migliora da sola con **Improver Agent**:

```
Ogni ciclo agenti → Improver analizza output → Learnings salvati → 
Definizioni aggiornate → Agenti migliori al ciclo successivo
```

**Meccanismo:**
- Learnings: `_fleet/learnings/{agent-id}-learnings.md`
- Improvement ticket: `_fleet/improvements/improvements-{data}.md`
- Trigger: automatico dopo `./_bin/agents` o manuale via `./_bin/improve`

**Regola per Sisyphus:** dopo ogni task delegato, valuta la qualità (1-5)
e lascia un learning in `_fleet/learnings/` — così gli agenti migliorano
a ogni iterazione.

## Regole per Sisyphus

1. **LEGGERE** `.agent.md`, `MANIFESTO.md` e `PROTOCOLLO.md` prima di interagire con la Memoria
2. **Se non è chiarissimo, CHIEDERE** — attivare FASE 0 del protocollo: domande mirate (dirette o indirette a seconda del caso), mai indovinare. Fornire esempi se utile.
3. **Preferire query script** (`query.py`) per cercare — non leggere file uno a uno
   - **Per spiegare all'utente**: usa `--human` (output in italiano semplice)
   - **Per agent/analisi tecnica**: usa output normale o `--json`
   - **Per leggere contenuto completo di un nodo**: usa `--leggi <id>`
4. **Leggere `_inbox/`** all'inizio di ogni sessione per vedere cosa gli agent hanno trovato
5. **Applicare il protocollo** di `PROTOCOLLO.md` anche per le tue scritture dirette — 7 gate + post-write
6. **Dopo aver aggiunto un nodo**, lanciare `./_bin/agents` per aggiornare tutto
7. **Promuovere suggerimenti** dalla inbox agli archi reali quando hanno senso
8. **Non modificare** `archi/relazioni.json` direttamente — usare il flusso suggerimento → approvazione
9. **Dopo ogni task delegato ad un agente**, valutare la qualità (1-5) e lasciare learning in `_fleet/learnings/`
10. **Verificare** che ogni scrittura passi `memoria_writer.py --validate-all` — errori non verbalizzati
11. **Non scrivere mai** ciò che un agente può scrivere — delega sempre quando possibile

## ⏱️ Protocollo Timeout & Recovery (regola #12)

> **Principio**: se un task delegato non torna in tempo, non aspettare — investiga e recupera.

### Tempi massimi per categoria

| Categoria | Timeout | Cosa fare se scade |
|-----------|---------|-------------------|
| `quick` | 3 min | Controlla, se fallito fallo direttamente |
| `visual-engineering` | 8 min | Controlla output parziale, completa tu |
| `general` | 5 min | Controlla, se fallito re-delega con più contesto |
| `ultrabrain` / `deep` | 12 min | Controlla, consulta Oracle prima di riprovare |
| `explore` / `librarian` | 3 min | Se non arriva, rifai la ricerca direttamente |

### Sequenza recovery

```
1. SCOPRI → heartbeat check <task_id>
   └── running? → aspetta ancora (max 2 check extra)
   └── done? → ✅ vai avanti
   └── timeout? → vai a #2
   └── failed? → vai a #3

2. SE TIMEOUT
   └── Leggi output parziale del task (background_output)
   └── Valuta: era quasi finito? → completa tu
   └── Era all'inizio? → rilancio con istruzioni più semplici
   └── Troppo complesso? → spezza in 2 task + fallback diretto

3. SE FALLITO
   └── Leggi errore
   └── Se istruzioni poco chiare → rilancio con prompt più semplice
   └── Se errore tecnico → fixa tu
   └── Se 2 tentativi falliti → fallo direttamente

4. REGOLA D'ORO: dopo 2 tentativi falliti, SMETTI di delegare.
   Fai direttamente. Il tempo perso a riprovare supera il tempo di farlo.
```

### Tracciamento

```bash
./_bin/heartbeat start <task_id> "descrizione" <timeout_sec> [fallback]
./_bin/heartbeat check <task_id>
./_bin/heartbeat done <task_id>
./_bin/heartbeat fail <task_id> "motivo"
```
